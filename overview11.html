Granite as a 10-Tuple — Set Theory Meets Geology

An intuitive slide that connects ordered tuples to rock composition

What the slide says

We compare different granite samples.

Each granite contains minerals like: quartz, K-feldspar, albite, biotite, muscovite.

To describe one granite completely, we need to know:

  • The amount of each mineral (e.g. % quartz, % feldspar, …)
  • The characteristic grain size of each mineral (how big the crystals are)

Altogether, there are 10 variables (5 minerals × 2 properties each = 10).

These can be seen as an ordered 10-tuple (like a 10-item list).

Breaking it down

In set theory, a tuple (like (a,b,c,…)) captures order and position.

Unlike a plain set (where order doesn’t matter), in an ordered tuple the 1st, 2nd, … positions matter.

So a granite sample can be represented as:

(quartz amount, feldspar amount, …, muscovite amount, quartz size, feldspar size, …, muscovite size)

That’s 10 slots in total.

Why this matters

  • This shows how set theory and ordered tuples describe real-world data like rock composition.
  • Each granite is not just “a set of minerals” but a structured collection of properties.
  • Geologists can compare granites scientifically: two granites are different if any of the 10 values differ.
