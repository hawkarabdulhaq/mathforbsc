Assignment — Functions & Relations (Porosity Grid)

Feladat — Függvények & Relációk (Porozitás rács)

Context / Háttér

EN: Use the Porosity Grid controls: Load Measurements M, Clear All, Single-valued mode, Active Domain A (checkboxes), Save Solution (JSON). “Function on active A? ✓” checks the row rule: each checked depth has exactly one φ. Range size = number of unique φ values in the current selection. The grid shows A×T (T ⊂ B).

HU: Használd a Porozitás rács vezérlőit: M betöltése, Mind törlése, Egyértékes mód, Aktív tartomány A (jelölőnégyzetek), Megoldás mentése (JSON). A „Függvény az aktív A-n? ✓” a sor-szabályt vizsgálja: minden bejelölt mélységhez pontosan egy φ tartozik. Az Értékkészlet mérete = az aktuális kiválasztásban szereplő különböző φ-k száma. A rács az A×T-t mutatja (T ⊂ B).

Hint: EN: Use dot for decimals (e.g. 0.24). • HU: Tizedeshez pontot használj (pl. 0,24 helyett 0.24).

Questions / Kérdések (10 items)
  1. 1) Single-valued mode behavior
    EN: Single-valued mode is ON. You click φ=0.18 at 1000 m, then φ=0.22 in the same row. What happens?
    HU: Egyértékes mód BE. Előbb φ=0.18-ra kattintasz 1000 m-nél, majd ugyanabban a sorban φ=0.22-re. Mi történik?
    A) Both stay selected. / Mindkettő bent marad.
    B) The new click replaces the old; the row keeps exactly one φ. / Az új kiválasztás felülírja a régit; a sorban pontosan egy φ marad.
    C) Both are cleared. / Mindkettő törlődik.
    D) Nothing changes. / Semmi sem változik.

    Expected: B

  2. 2) Active Domain and Function ✓
    EN: You uncheck 1150 m in Active Domain A. Does that depth still count in “Function on active A?”
    HU: Kiveszed a pipát az 1150 m-ből az Aktív A listában. Beleszámít még a „Függvény az aktív A-n?” ellenőrzésbe?

    Expected: No

  3. 3) Load Measurements M
    EN: Load Measurements M (below). In 1–2 sentences: is M a function on active A? Why?
    HU: Töltsd be az M méréseket (lent). 1–2 mondatban: függvény-e M az aktív A-n? Miért?
    A = [980,...,1203]
M = (980,0.16), (1000,0.18), (1001,0.22), (1002,0.21), (1010,0.20),
    (1050,0.23), (1100,0.24), (1150,0.20), (1198,0.19), (1203,0.21)

    Expected: Yes, each depth appears once with one φ (single-valued).

  4. 4) Range size from M
    EN: Using M above, how many unique φ values are in Range(M)?
    HU: A fenti M alapján hány különböző φ érték van az Értékkészletben?

    Expected: 8

  5. 5) List Range(M)
    EN: List Range(M) in ascending order (comma-separated).
    HU: Írd fel az Értékkészletet (M) növekvő sorrendben, vesszővel elválasztva.

    Expected: 0.16, 0.18, 0.19, 0.20, 0.21, 0.22, 0.23, 0.24

  6. 6) Single-valued OFF duplicate row
    EN: Single-valued mode is OFF. You click (1001,0.22) and (1001,0.17). Is your selection a function? Why?
    HU: Egyértékes mód KI. Rákattintasz (1001,0.22) és (1001,0.17). Függvény-e így a kijelölés? Miért?

    Expected: No — same input (1001) maps to two outputs; violates rule.

  7. 7) What does the green ✓ guarantee?
    EN: What does a green ✓ next to “Function on active A?” guarantee?
    HU: Mit garantál a zöld ✓ a „Függvény az aktív A-n?” mellett?
    A) Each checked depth has exactly one φ. / Minden bejelölt mélységhez pontosan egy φ tartozik.
    B) Range size equals |A|. / Az értékkészlet mérete = |A|.
    C) Values come only from M. / Az értékek csak M-ből származnak.
    D) φ increases with depth. / A φ nő a mélységgel.

    Expected: A

  8. 8) Range size interpretation
    EN: After selecting R, UI shows “Range size: 9”. In 2 sentences: what does this tell you, and what not?
    HU: Az R kijelölése után a UI „Értékkészlet mérete: 9” üzenetet ír. Két mondatban: mit jelent ez, és mit nem?
    R = (980,0.15), (1000,0.16), (1001,0.17), (1002,0.18), (1010,0.19),
    (1050,0.20), (1100,0.21), (1150,0.22), (1198,0.22), (1203,0.23)

    Expected: 9 distinct φ values; tells distinctness, not function/trend.

  9. 9) A×T size
    EN: With |A|=10 and |T|=10, how many cells are in A×T?
    HU: Ha |A|=10 és |T|=10, hány cella van az A×T rácsban?

    Expected: 100

  10. 10) Save JSON & verify function
    EN: “Save Solution (JSON)” exports your selection. Name two fields expected in JSON (e.g. active domain, selected pairs) and how to verify depth→φ is a function.
    HU: A „Megoldás mentése (JSON)” exportálja a kijelölést. Nevezz meg két mezőt a JSON-ban (pl. aktív tartomány, kiválasztott párok) és írd le, hogyan ellenőrzöd, hogy a mélység→φ függvény.

    Expected: Fields like active_domain & selected_pairs; each depth appears once with one φ.
