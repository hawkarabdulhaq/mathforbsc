2) Tiny porosity log (our data)

# Depths (domain A) and possible porosities (codomain B)
A = {1198, 1200, 1203}
B = {0.19, 0.21, 0.24}

# The measurements (a relation = set of true pairs)
M = {(1198, 0.19), (1200, 0.24), (1203, 0.21)}
  • M is a relation because it’s some pairs from all possibilities (A × B).
  • In words: “at 1198 m → 0.19; at 1200 m → 0.24; at 1203 m → 0.21.”

2) Mini porozitás‑napló (adataink)

# Értelmezési tartomány (A) és kódomain (B)
A = {1198, 1200, 1203}
B = {0.19, 0.21, 0.24}

# Mérések (reláció = igaz párok halmaza)
M = {(1198, 0.19), (1200, 0.24), (1203, 0.21)}
  • M egy reláció, mert az összes lehetőségből (A × B) néhány pár igaz.
  • Szavakban: „1198 m → 0.19; 1200 m → 0.24; 1203 m → 0.21”.

3) When is a relation a function?

If each input depth appears once with one output. In code, think a dictionary:

phi = {1198: 0.19, 1200: 0.24, 1203: 0.21}   # function: porosity at depth
phi[1200]  # -> 0.24

This passes the “one depth → one porosity” rule (like the vertical‑line test).

Not a function example (conflicting outputs for the same depth):

M_bad = {(1200, 0.22), (1200, 0.24)}  # two outputs for 1200 -> not a function

3) Mikor függvény egy reláció?

Ha minden bemenet (mélység) pontosan egyszer szerepel egy kimenettel. Kódban ez egy szótár:

phi = {1198: 0.19, 1200: 0.24, 1203: 0.21}   # függvény: porozitás mélységhez
phi[1200]  # -> 0.24

Ez megfelel az „egy mélység → egy porozitás” szabálynak (mint a „függőleges egyenes” próba).

Nem függvény példa (ugyanahhoz a mélységhez több kimenet):

M_bad = {(1200, 0.22), (1200, 0.24)}  # két kimenet 1200-hoz -> nem függvény

4) Domain, Codomain, Range (super short)

  • Domain A = {1198, 1200, 1203} (depths you put in).
  • Codomain B = allowed porosity numbers (here {0.19, 0.21, 0.24}, but in science usually [0…1]).
  • Range = {0.19, 0.24, 0.21} (values you actually got).
domain   = set(phi.keys())    # {1198, 1200, 1203}
range_   = set(phi.values())  # {0.19, 0.21, 0.24}
codomain = B                   # allowed porosity values

4) Értelmezési tartomány, kódomain, képhalmaz (röviden)

  • A tartomány = {1198, 1200, 1203} (beadott mélységek).
  • B kódomain = megengedett porozitások (itt {0.19, 0.21, 0.24}, általában [0…1]).
  • Képhalmaz = {0.19, 0.24, 0.21} (a ténylegesen kapott értékek).
domain   = set(phi.keys())    # {1198, 1200, 1203}
range_   = set(phi.values())  # {0.19, 0.21, 0.24}
codomain = B                   # megengedett porozitásértékek

5) “Any” and “All” checks (quantifiers made easy)

“There exists a depth with porosity ≥ 0.25?” → any(...)

any(v >= 0.25 for v in phi.values())  # False on our data

“For all depths, porosity ≤ 0.30?” → all(...)

all(v <= 0.30 for v in phi.values())  # True on our data

Think: any = some, all = every.

5) „Van‑e” és „minden” ellenőrzések (kvantorok egyszerűen)

„Létezik‑e olyan mélység, ahol porozitás ≥ 0.25?” → any(...)

any(v >= 0.25 for v in phi.values())  # Hamis ezekre az adatokra

„Minden mélységre porozitás ≤ 0.30?” → all(...)

all(v <= 0.30 for v in phi.values())  # Igaz ezekre az adatokra

Gondolat: any = van olyan, all = mindegyik.

6) One other relation (depth vs depth)

“Shallower than” is a relation (not a function):

R_shallow = {(x, y) for x in A for y in A if x < y}
# {(1198,1200), (1198,1203), (1200,1203)}

6) Egy másik reláció (mélység vs. mélység)

A „sekélyebb mint” egy reláció (nem függvény):

R_shallow = {(x, y) for x in A for y in A if x < y}
# {(1198,1200), (1198,1203), (1200,1203)}
